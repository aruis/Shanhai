# 半三维动态生态原胞自动机 Eco-CA 系统设计说明书

## 1. 设计目标

Eco-CA 是一个时间、空间、状态完全离散的生态动力系统。系统以二维正方形网格为载体，通过局部水文、季节脉冲、养分循环、植物竞争和动物本能行为，模拟一个能够自组织出河流、湖泊、草场、森林、迁徙、隔离和周期性种群波动的生态圈。

本设计坚持以下原则：

- 所有物理、生物行为都由局部规则驱动。
- 动物不拥有全局视野，只能基于周围 8 格和局部环境信号决策。
- 宏观现象不直接编码，而是由水、养分、植物、动物之间的反馈涌现。
- 数值更新采用同步 tick，避免遍历顺序影响仿真结果。

## 2. 空间拓扑

系统使用二维正方形网格 `Grid[width][height]`。每个原胞使用多层矩阵叠加存储状态。

邻居定义为摩尔邻居：

- 每个格子最多有 8 个邻居。
- 边界格子按地图边界裁剪。
- 所有扩散、流动、播种、移动默认只作用于这 8 个邻居。

## 3. 初始化协议

Eco-CA 对初始条件敏感，因此必须把初始化视为设计规格的一部分，而不是实现细节。

每次仿真必须记录：

- 地图尺寸。
- 随机种子 `seed`。
- 地形生成算法和参数。
- 泉眼数量与位置。
- 初始水量、湿度、养分、植物和动物分布。
- 参数 preset 名称。

### 3.1 地形初始化

推荐支持两类地图：

- `scenario`：用于测试的固定模板地图。
- `procedural`：用于正常游玩的随机生成地图。

固定模板至少包括：

- 斜坡入海：验证水流和海洋泄洪。
- 封闭盆地：验证湖泊生成和稳定。
- 盆地缺口：验证湖泊外溢和出湖河。
- 山脉分割平原：验证动物地缘隔离。
- 河谷平原：验证河谷草场和种群增长。

随机地图生成应满足：

- 海洋至少与地图边缘连通。
- 高山区域可生成泉眼。
- 平地必须存在足够大的连通区域，供动物种群初始化。
- 中高山和水系可以切割平原，但不应默认把所有平地切成过小碎片。

### 3.2 水源初始化

泉眼从 `H = 4` 的高山格子中选择。

若地图没有高山，则可以使用测试模式的人工泉眼，但应在场景元数据中显式标记。

泉眼数量推荐由地图面积决定：

```text
spring_count = clamp(area / spring_area_factor, spring_min, spring_max)
```

### 3.3 生态初始化

初始状态推荐：

- `W = 0`，除非场景显式预置河流或湖泊。
- `M` 根据海拔、距水源距离或噪声初始化。
- `N` 在平地和低山较高，中高山较低。
- 草本优先投放在湿度和养分较高的平地。
- 木本优先投放在低山和山麓。
- 动物投放在平地连通域中，且每个连通域可单独配置投放数量。

动物投放不应直接放在 `RIVER`、`LAKE`、`OCEAN`、`ICE` 或 `H >= 3` 的格子上。

## 4. 状态层定义

### 4.1 静态地貌层

#### Height Matrix `H`

`H` 是静态整数矩阵，表示基础海拔。

| 值 | 含义 |
| --- | --- |
| `0` | 海平面 |
| `1` | 平地 |
| `2` | 低山 |
| `3` | 中山 |
| `4` | 高山 |

`H` 不随仿真过程变化。

#### Base Terrain Matrix `B`

`B` 是静态或极低频变化的基础地貌层，用于避免把地貌和水体覆盖混在同一层。

| 值 | 含义 |
| --- | --- |
| `OCEAN` | 海洋，通常对应 `H = 0` |
| `PLAIN` | 平地，通常对应 `H = 1` |
| `LOW_HILL` | 低山，通常对应 `H = 2` |
| `MID_MOUNTAIN` | 中山，通常对应 `H = 3` |
| `HIGH_MOUNTAIN` | 高山，通常对应 `H = 4` |

### 4.2 动态地表覆盖层

#### Surface Matrix `S`

`S` 表示当前格子的动态地表覆盖。

| 值 | 含义 |
| --- | --- |
| `DRY` | 普通可生长陆地 |
| `WET` | 潮湿地表 |
| `RIVER` | 河流 |
| `LAKE` | 湖泊 |
| `ICE` | 冬季冻结水面 |
| `BARREN` | 荒漠化或贫瘠地 |

`S` 是水文、季节、荒漠化共同作用的结果。

### 4.3 水文层

#### Surface Water Matrix `W`

`W >= 0`，表示地表积水或当前格子的可流动水量。

用途：

- 河流流动。
- 湖泊蓄水。
- 洪水扩散。
- 海洋泄洪。

#### Soil Moisture Matrix `M`

`M >= 0`，表示土壤可用湿度。

用途：

- 植物吸水。
- 动物通过局部湿度间接寻找水源。
- 水边生态带生成。

`W` 与 `M` 必须区分：地表没有积水的格子也可能拥有足够土壤湿度。

#### Flow Matrix `F`

`F >= 0`，表示最近一个 tick 或短期窗口内流经该格子的水量。

用途：

- 判断长期河道。
- 区分流动水和静止水。
- 形成河谷生态带。

### 4.4 养分层

#### Nutrient Matrix `N`

`N >= 0`，表示土壤肥沃度。

用途：

- 支持植物生长。
- 接收枯死植物和动物尸体回流。
- 参与荒漠化和生态恢复。

### 4.5 植物层

#### Plant Matrix `P`

每个格子最多持有一个主导植物状态。

植物状态包含：

- `type`: `EMPTY | HERB | WOODY`
- `maturity`: 成熟度
- `biomass`: 生物量
- `stress`: 干旱或贫瘠压力累计
- `dormant`: 是否冬眠

`biomass` 是死亡返还养分的基础。

### 4.6 动物层

#### Animal Matrix `A`

每个格子可容纳多个小型草食动物个体，但应设置承载上限 `K_cell`。

动物个体状态包含：

- `energy`: 当前能量
- `energy_max`: 能量上限
- `thirst`: 当前水分
- `thirst_max`: 水分上限
- `age`: 年龄
- `sex`: 性别
- `reproduce_cooldown`: 繁殖冷却
- `alive`: 是否存活

动物没有全局地图、全局寻路或远程扫描能力。

## 5. 全局时间与季节

系统维护全局 tick：

```text
tick = 0, 1, 2, ...
```

每 `T_season` 个 tick 切换一次季节。

```text
春季 Spring -> 夏季 Summer -> 秋季 Autumn -> 冬季 Winter -> 春季 Spring
```

### 5.1 春季

- 高山泉眼开启。
- 高海拔积雪或冬季储水开始释放。
- 土壤湿度恢复速度提高。
- 草本进入快速萌发阶段。
- 动物从冬季低活跃状态恢复。

### 5.2 夏季

- 泉眼维持较高出水。
- 水文扩张能力最强。
- 草本繁殖概率最高。
- 动物进食和繁殖活跃。

### 5.3 秋季

- 泉眼出水下降。
- 植物生长速度放缓。
- 草本逐步进入衰退。
- 动物能量上限临时提高，进入脂肪积累阶段。

### 5.4 冬季

- 泉眼关闭。
- 河流和湖泊可进入冻结或低流动状态。
- 草本强制死亡或大规模枯萎。
- 木本进入冬眠。
- 动物依靠秋季储能和树木庇护存活。

## 6. Tick 更新阶段

每个 tick 采用同步更新。所有阶段先读取当前状态，写入下一状态或中间缓冲，最后统一提交。

推荐顺序：

1. 季节参数更新。
2. 泉眼、降水、融雪等水源注入。
3. 水文出流计算。
4. 水文汇总到 `W_next` 和 `F_next`。
5. 海洋吞噬、蒸发、渗漏、冻结或融化。
6. 土壤湿度 `M` 更新。
7. 河流、湖泊、潮湿地表判定。
8. 养分 `N` 更新。
9. 植物生长、死亡、繁殖。
10. 动物代谢、饮水、进食、移动、繁殖、死亡。
11. 荒漠化与生态恢复判定。
12. 统计指标采样。

## 7. 水文规则

### 7.1 水源注入

泉眼是固定或半固定格子，通常位于 `H = 4` 的高山区域。

每 tick 注入量由季节决定：

| 季节 | 泉眼出水 |
| --- | --- |
| 春季 | `spring_output_high` |
| 夏季 | `summer_output_high` |
| 秋季 | `autumn_output_low` |
| 冬季 | `0` |

### 7.2 有效水位

每个格子的有效水位：

```text
water_level(cell) = H(cell) + W(cell) * water_depth_scale
```

水倾向流向有效水位更低的邻居。

所有水文计算必须满足单 tick 质量平衡：

```text
W_next =
    W_current
  + source
  + inflow
  - outflow
  - evaporation
  - seepage
  - ocean_sink
```

其中：

- `source` 来自泉眼、降水、融雪等外源输入。
- `inflow` 是邻居流入当前格子的水。
- `outflow` 是当前格子流向邻居的水。
- `evaporation` 是蒸发损耗。
- `seepage` 是渗入土壤或地下的损耗。
- `ocean_sink` 是流入海洋后从系统移除的水。

除 `source` 和 `ocean_sink` 等显式外部项外，水量不得凭空增减。

### 7.3 出流计算

对每个非海洋格子，先计算出流意图，再统一汇总：

1. 找到所有可接收水的邻居。
2. 邻居必须满足：
   - 不是 `OCEAN`
   - 有效水位低于当前格子，或在平地扩散阶段允许近似等水位扩散
3. 根据水位差分配出流比例。
4. 当前格子的总出流不得超过 `max(0, W(cell) - W_reserve) * max_outflow_ratio`。

出流写入缓冲，不直接修改邻居当前值。

推荐公式：

```text
available = max(0, W(cell) - W_reserve)
outflow_budget = available * max_outflow_ratio

delta_i = max(0, water_level(cell) - water_level(neighbor_i))
outflow_i = outflow_budget * delta_i / sum(delta)
```

若不存在更低有效水位邻居，但存在近似等水位邻居，可进入平地扩散：

```text
if abs(water_level(cell) - water_level(neighbor_i)) <= flat_epsilon:
    candidate_i = neighbor_i
```

平地扩散只能使用更小比例：

```text
flat_outflow_budget = available * flat_diffusion_ratio
```

这样可以避免平地水面无限振荡。

### 7.4 海洋吞噬

流入 `B = OCEAN` 的水在本 tick 结束时清零。

海洋是系统水量的泄洪口。

若当前格子本身为 `OCEAN`：

```text
W_next = 0
M_next = ocean_moisture_baseline
```

海洋格子不参与植物、动物和土壤养分更新。

### 7.5 蒸发与渗漏

每个格子的损耗由以下因素决定：

- 季节。
- 地表覆盖。
- 是否为湖泊。
- 湖泊面积。
- 土壤类型或地貌。

湖泊可以保留面积惩罚机制，但应解释为大面积浅水带来的渗漏和蒸发增强：

```text
if lake_component_size > S_lake_soft_cap:
    evaporation_rate *= lake_area_penalty
    seepage_rate *= lake_area_penalty
```

蒸发和渗漏必须受当前水量限制：

```text
evaporation = min(W, evaporation_rate * seasonal_evap_multiplier)
seepage = min(W - evaporation, seepage_rate * terrain_seep_multiplier)
```

渗漏的一部分可以转化为土壤湿度：

```text
M_next += seepage * seep_to_moisture_ratio
```

### 7.6 河流生成

河流由长期方向性流量形成。

对每个陆地格子维护短期或长期流量累计：

```text
flow_memory = decay(flow_memory) + F_current
```

若满足：

- `B` 不是 `OCEAN`
- `S` 不是 `LAKE`
- `flow_memory > W_river`
- 持续 `river_form_ticks`

则 `S = RIVER`。

`RIVER` 格子具有更低流动阻力和更高水分外溢能力。

`flow_memory` 必须带衰减：

```text
flow_memory_next = flow_memory * flow_memory_decay + F_current
```

若长期无水流经过：

```text
if flow_memory < river_decay_threshold for river_dry_ticks:
    S = DRY or WET
```

这样河道可以在长期枯水后退化，但不会因一轮洪水永久变成河流。

### 7.7 湖泊生成

湖泊由长期静止水形成。

河流看流量，湖泊看蓄水，二者不应仅用同一个 `W` 阈值判定。

推荐维护静水记忆：

```text
standing_water_memory_next =
    standing_water_memory * standing_water_decay
  + max(0, W - outflow)
```

湖泊形成条件改为：

- `standing_water_memory > lake_memory_threshold`
- `outflow / max(W, epsilon) < lake_outflow_ratio_threshold`
- 持续 `lake_form_ticks`

湖泊退化条件：

- `W < lake_min_water`
- 持续 `lake_dry_ticks`

退化后：

```text
S = WET or DRY
```

### 7.8 湖泊外溢与水系串联

湖泊连通域水位过高时：

1. 找到湖泊边界邻居中有效水位最低的陆地格子。
2. 向该方向泄洪。
3. 被长期泄洪的格子可能转化为 `RIVER`。

该机制允许形成：

```text
高山泉眼 -> 山谷河流 -> 湖泊 -> 出湖河流 -> 海洋
```

湖泊连通域应作为整体计算外溢。

对每个湖泊连通域：

1. 计算连通域平均或总水量。
2. 计算湖泊边界候选格。
3. 找到边界中 `water_level` 最低且可通行的陆地格子。
4. 若湖泊水位高于该边界格超过 `lake_spill_epsilon`，则发生外溢。

外溢量：

```text
spill = min(lake_spill_max, (lake_level - boundary_level) * lake_spill_rate)
```

外溢方向写入 `F`，长期外溢可把边界路径转化为 `RIVER`。

## 8. 土壤湿度规则

`M` 是植物和动物局部感知的关键环境场。

### 8.1 湿度来源

- 地表水 `W` 向土壤渗透。
- `RIVER` 和 `LAKE` 对周边格子提供湿度扩散。
- 春季融雪或降雨提高全局恢复。

### 8.2 湿度损耗

- 季节性蒸发。
- 植物吸水。
- 荒漠化地表加速流失。

### 8.3 湿度扩散

湿度可以向邻居缓慢扩散，并随距离形成局部梯度。

动物不需要看到远处水源，只需在口渴时偏向更高 `M`、更低地势或更高植被密度的邻格。

## 9. 养分规则

### 9.1 养分增加

养分来源：

- 植物死亡后按 `biomass` 回流。
- 动物死亡后按剩余能量和体质量回流。
- 河流和湖泊周边灌溉沉积。
- 春季微量自然恢复。

### 9.2 养分减少

养分消耗：

- 草本生长。
- 木本生长。
- 生态恢复过程。

### 9.3 养分守恒倾向

除自然恢复和海洋泄出等外部项外，生物死亡应尽量把生物量返还到当前格子或附近格子。

## 10. 植物规则

### 10.1 草本 `HERB`

草本是一年生、快速生长、高繁殖、低储备植物。

#### 生存条件

- `H in [1, 2]`
- `S not in [RIVER, LAKE, ICE]`
- `M >= herb_min_moisture`
- `N >= herb_min_nutrient`

#### 生长

春夏生长最快，秋季减慢，冬季死亡。

生长消耗：

- 少量 `N`
- 中等或较高 `M`

#### 死亡

草本死亡条件：

- 冬季强制死亡。
- 长期干旱。
- 长期养分不足。
- 被动物吃掉。

死亡后按 `biomass * herb_nutrient_return` 增加当前格子的 `N`。

#### 播种

成熟草本在春夏向邻居空格播种。

播种成功率受以下因素影响：

- 目标格子的 `M`
- 目标格子的 `N`
- 季节
- 下坡加成
- 是否被水体阻隔

从 `H = 2` 向 `H = 1` 下坡播种时成功率提高。

草本种子不能跨越 `RIVER`、`LAKE`、`OCEAN`。

### 10.2 植物竞争与同格关系

默认实现采用“单主导植物”模型，但需要显式定义草本与木本的替换关系。

推荐规则：

- `EMPTY -> HERB`：草本播种成功后占据空格。
- `EMPTY -> WOODY`：木本播种成功后生成木本幼苗。
- `HERB -> WOODY`：木本幼苗可在草本格上建立，但成功率受草本生物量抑制。
- `WOODY -> HERB`：草本不能替换存活木本。
- `WOODY dead -> EMPTY or HERB-ready`：木本死亡后返还养分，并转为空地或高养分空地。

如果实现需要林下草，可以升级为双生物量模型：

```text
plant_layer = {
    herb_biomass,
    woody_biomass,
    woody_maturity
}
```

第一版推荐仍使用单主导植物，避免植物层复杂度过早膨胀。

木本成熟度影响：

- 庇护效果。
- 播种概率。
- 养分消耗。
- 草本替换抑制。

### 10.3 木本 `WOODY`

木本是多年生、慢生长、低繁殖、高稳定植物。

#### 生存条件

- `H in [1, 2, 3]`
- `S not in [RIVER, LAKE, ICE]`
- `N` 长期不为零

#### 生长

木本消耗较高 `N`，但对 `M` 的需求低于草本。

#### 冬眠

冬季木本进入冬眠：

- 不生长。
- 不繁殖。
- 大幅降低或停止消耗 `M` 和 `N`。
- 为动物提供庇护。

#### 繁殖

木本繁殖概率低，但具备特殊扩散能力：

- 可跨越 1 格宽 `RIVER` 或 `LAKE`。
- 可从低海拔向更高海拔扩散，例如 `H = 2 -> H = 3`。
- 可逐步形成低山和中山林带。

## 11. 动物规则

系统默认只保留一种小型平原草食动物。

动物没有全局视野。每 tick 只能基于：

- 当前格子状态。
- 8 个邻居状态。
- 自身能量、水分、年龄和繁殖冷却。
- 局部环境信号，如 `M`、植物、地势、树木庇护。

### 11.1 可通行地形

动物可以进入：

- `H in [1, 2]`
- `S not in [RIVER, LAKE, ICE]`
- `B not OCEAN`

动物不能进入：

- `H >= 3`
- `RIVER`
- `LAKE`
- `OCEAN`
- `ICE`

### 11.2 移动消耗

基础移动消耗：

```text
move_cost = E_base
```

上坡惩罚：

```text
if H_next > H_current:
    move_cost *= uphill_penalty
```

默认：

```text
uphill_penalty = 4
```

下坡不额外增加消耗。

### 11.3 行为优先级

动物每 tick 选择一个动作，优先级如下。

动物阶段必须拆成同步子阶段执行。每个子阶段读取上一子阶段提交后的稳定状态，或读取本 tick 开始时的快照，不能在遍历个体时直接修改共享资源。

推荐子阶段：

1. 代谢扣除。
2. 死亡标记。
3. 行动意图生成。
4. 移动仲裁。
5. 饮水和进食结算。
6. 繁殖结算。
7. 死亡返肥提交。

#### 0. 代谢扣除

每个存活动物先扣除基础代谢：

```text
energy -= base_metabolism
thirst -= thirst_decay
age += 1
reproduce_cooldown = max(0, reproduce_cooldown - 1)
```

冬季额外扣除：

```text
if season == Winter:
    energy -= winter_energy_cost
```

若当前格子有成熟 `WOODY`：

```text
effective_winter_energy_cost = winter_energy_cost * woody_shelter_cost_multiplier
```

`woody_shelter_cost_multiplier` 应小于 1。

#### 1. 生理死亡判定

若 `energy <= 0` 或 `thirst <= 0`，动物死亡。

死亡返肥量：

```text
nutrient_return = animal_body_nutrient_return
```

死亡个体先进入 `dead_animals_buffer`，返肥在子阶段 7 统一提交。

#### 2. 寻水本能

当 `thirst < thirst_critical`：

1. 若当前格子邻接 `RIVER` 或 `LAKE`，补满水分。
2. 否则在合法邻居中选择局部水分信号最高者。

水分信号可以由以下因素组成：

```text
water_signal =
    M * moisture_weight
  + adjacent_water_bonus
  + downhill_bonus
  + vegetation_density_bonus
```

这保持了 8 格视野，同时允许动物顺着环境梯度接近远处水源。

寻水阶段只生成移动意图或饮水意图：

- 若当前格子邻接水体，生成 `DRINK` 意图。
- 否则生成 `MOVE(target_cell)` 意图。

不会立即修改 `thirst` 或位置。

#### 3. 寻食本能

当不口渴且 `energy < hunger_threshold`：

- 优先移动到有 `HERB` 的合法邻居。
- 若当前格子有 `HERB`，直接进食。
- 进食后草本消失或生物量降低，动物能量增加。

若多只动物同时选择同一格草本，使用草本 `biomass` 作为可分配资源：

```text
food_share = herb_biomass / eater_count
energy_gain = min(food_share * herb_energy_factor, energy_max - energy)
```

若草本生物量被吃到低于 `herb_min_biomass`，该格 `P = EMPTY`。

#### 4. 秋季脂肪积累

秋季时：

- `energy_max` 临时提高到 `2 * base_energy_max`。
- 动物在不口渴时提高进食优先级。
- 目标是尽量在冬季前积累能量。

#### 5. 冬季避护

冬季时：

- 草本大多死亡，食物稀缺。
- 动物每 tick 扣除基础能量。
- 若当前格子有 `WOODY`，冬季能量消耗降低。
- 若邻居有 `WOODY`，动物偏向进入该格子。

推荐把冬季进林写成局部偏好，而不是远程寻找：

```text
if season == Winter and adjacent WOODY exists:
    move toward WOODY with high probability
```

宏观上的“进山避冬”由林带分布、随机游荡和死亡筛选共同产生。

该规则仍然只看 8 格。动物不能远程知道森林位置。

#### 6. 繁殖

繁殖条件：

- 成年。
- `energy` 高于繁殖阈值。
- `thirst` 高于繁殖阈值。
- `reproduce_cooldown = 0`。
- 当前格子或邻居存在异性成年个体。
- 周围存在未超过承载上限的合法格子。

繁殖结果：

- 在合法邻格生成幼崽。
- 双亲进入繁殖冷却。
- 双亲消耗一定能量。

繁殖在移动和进食之后结算。若多个繁殖意图争夺同一出生格：

1. 按固定随机种子生成稳定洗牌顺序。
2. 逐个检查目标格承载上限。
3. 超出上限的繁殖意图失败。

繁殖失败不生成幼崽，但可以选择不扣除双亲能量。

#### 7. 健康游荡

若无紧急需求：

- 在合法邻居中随机游荡。
- 可轻微偏向草本更多、湿度更高、树木更近的格子。

### 11.4 移动仲裁

所有动物先生成移动意图，再统一仲裁。

每个目标格的可用容量：

```text
capacity_left = K_cell - current_animals_staying - accepted_incoming
```

仲裁规则：

1. 对所有移动意图按 `hash(seed, tick, animal_id)` 生成稳定顺序。
2. 若目标格合法且仍有容量，则移动成功。
3. 若目标格已满或变为非法，则移动失败，动物留在原地。
4. 移动成功后扣除移动能量。

该规则允许随机性，但保证相同 seed 下完全可复现。

### 11.5 局部视野原则

动物始终只看 8 格。系统不为动物提供全局水源、森林或路径。

远距离趋向行为应优先通过环境场产生：

- 湿度梯度。
- 草本沿河谷分布。
- 地势低处更湿。
- 林带提供冬季存活优势。
- 死亡筛选改变幸存种群的空间分布。

只有当仿真结果显示动物行为过于随机或灭绝率不可控时，才考虑加入极弱记忆，例如：

- `last_water_direction`
- `last_safe_cell`
- `last_shelter_direction`

记忆不是基础规则的一部分。

## 12. 荒漠化与恢复

### 12.1 荒漠化

荒漠化不使用 `N = 0` 且 `M = 0` 的硬等值判断，而使用压力累计。

每个可荒漠化格子维护：

```text
dry_stress >= 0
```

当湿度和养分低于阈值时：

```text
dry_stress += dry_stress_gain * season_dry_multiplier
```

当湿度和养分恢复时：

```text
dry_stress -= dry_stress_recovery
```

若满足：

- `S = DRY`
- `M < barren_moisture_threshold`
- `N < barren_nutrient_threshold`
- `dry_stress > barren_enter_threshold`

则转为：

```text
S = BARREN
```

`BARREN` 格子难以自然长出植物。

### 12.2 恢复

荒漠化不应完全不可逆。

恢复同样使用累计分数：

```text
recovery_score >= 0
```

若 `BARREN` 格子满足：

- `M > recovery_moisture`
- `N > recovery_nutrient`
- 周围存在植物源

则：

```text
recovery_score += recovery_gain
```

否则：

```text
recovery_score -= recovery_decay
```

若：

```text
recovery_score > barren_exit_threshold
```

则恢复为：

```text
S = DRY
```

这允许生态退化和恢复循环。

`barren_enter_threshold` 必须高于 `barren_exit_threshold` 对应的恢复要求，形成滞回，避免 `DRY` 和 `BARREN` 高频闪烁。

## 13. 冲突解决规则

同一 tick 中多个规则可能竞争同一格子，推荐优先级如下：

1. 基础地貌限制：海洋、高山、不可通行规则优先。
2. 水文覆盖：湖泊、河流、冻结覆盖优先于植物。
3. 植物死亡先于植物新生。
4. 动物死亡先于繁殖。
5. 动物进食先于植物扩散提交。
6. 多个动物争夺同一目标格时，按随机顺序或稳定洗牌处理。
7. 格子超过承载上限时，触发迁移失败、竞争损耗或随机淘汰。

所有随机决策应使用可注入随机种子，保证仿真可复现。

## 14. 数值稳定性约束

Eco-CA 存在多个正反馈环，因此所有连续量都必须有上限、耗散或饱和函数。

### 14.1 连续量上限

以下变量必须定义局部上限：

| 变量 | 推荐上限 |
| --- | --- |
| `W` | `W_max` |
| `M` | `M_max` |
| `N` | `N_max` |
| `herb_biomass` | `herb_biomass_max` |
| `woody_biomass` | `woody_biomass_max` |
| 单格动物数量 | `K_cell` |
| 单个动物能量 | `energy_max` |
| 单个动物水分 | `thirst_max` |

任何更新后都应执行 clamp：

```text
value = clamp(value, 0, value_max)
```

### 14.2 饱和增长

资源驱动的增长不应使用无限线性增长。

推荐使用半饱和函数：

```text
resource_factor = resource / (resource + half_saturation)
growth = max_growth * resource_factor
```

适用对象：

- 植物吸收养分。
- 植物吸收湿度。
- 草本生长。
- 木本生长。
- 动物进食后的能量恢复。
- 土壤湿度对植物播种成功率的影响。

### 14.3 密度抑制

动物和植物繁殖需要密度抑制。

动物繁殖概率：

```text
birth_probability *= max(0, 1 - local_animal_count / K_cell)
```

草本播种概率：

```text
herb_spread_probability *= max(0, 1 - local_plant_density)
```

木本播种概率：

```text
woody_spread_probability *= max(0, 1 - woody_local_density)
```

### 14.4 外源输入与耗散

泉眼、春季恢复和降水是外源输入，必须被以下耗散项平衡：

- 海洋泄洪。
- 蒸发。
- 渗漏。
- 植物消耗。
- 动物代谢。
- 荒漠化损耗。

调参时应优先检查：

```text
external_input <= long_term_sink_capacity
```

若长期输入大于长期耗散，系统会趋向全图湿化或湖泊无限扩张。

## 16. 宏观涌现预期

### 16.1 水系自组织

高山泉眼、地势差、湖泊外溢和海洋泄洪共同生成稳定水系。

预期出现：

- 山谷溪流。
- 平原河道。
- 内陆湖泊。
- 出湖河。
- 河流入海口。

### 16.2 地缘隔离

中高山、海洋、河流和湖泊共同构成动物不可跨越屏障。

预期出现：

- 平原口袋中的局部种群。
- 被山脉和水系分割的生态岛。
- 某些区域局部灭绝，某些区域长期稳定。

### 16.3 河谷草场

河流和湖泊提高周边湿度与养分，促成草本爆发。

当前实现采用局部河岸富集规则：若一个非海洋、非水面陆地格邻接 `RIVER` 或 `LAKE`，每 tick 对该格的 `M` 与 `N` 施加小幅补给。补给写入下一状态缓冲后统一提交，因此不受遍历顺序影响。

预期出现：

- 河岸高生产力带。
- 春夏草场扩张。
- 动物向河谷聚集。
- 过度啃食后的局部退化。

### 16.4 低山森林庇护

木本能够向低山和中山扩散，并在冬季提供庇护。

预期出现：

- 低山林带。
- 平原草场与山麓林带分区。
- 冬季幸存个体集中在林缘或林内。

### 16.5 种群锯齿波

季节性资源脉冲和冬季死亡压力共同驱动周期性种群波动。

预期过程：

1. 春季草本爆发。
2. 夏季动物繁殖。
3. 秋季动物储能。
4. 冬季草本死亡，动物大量损耗。
5. 尸体和枯草返肥。
6. 次年春季草场再次爆发。

宏观曲线表现为动物数量的周期性 V 型或锯齿形波动。

## 17. 实现路线

### 阶段一：地形与水文

实现：

- `H`
- `B`
- `S`
- `W`
- `F`
- 泉眼
- 流动
- 蒸发
- 河流
- 湖泊

目标：

- 水系能稳定生成。
- 河流和湖泊不会无限扩张或瞬间干涸。

### 阶段二：湿度、养分与植物

实现：

- `M`
- `N`
- 草本
- 木本
- 季节性生长和死亡

目标：

- 河谷草场自然出现。
- 低山森林逐步形成。
- 冬季草本死亡后养分回流。

### 阶段三：动物代谢与移动

实现：

- 动物能量。
- 动物水分。
- 8 格局部移动。
- 饮水。
- 进食。
- 死亡返肥。

目标：

- 动物能在局部环境信号下聚集到水草丰盛区域。
- 不引入全局寻路。

### 阶段四：繁殖与冬季策略

实现：

- 年龄。
- 性别。
- 繁殖冷却。
- 秋季储能。
- 冬季树木庇护。

目标：

- 形成季节性种群涨落。
- 出现局部灭绝和幸存生态口袋。

## 18. 观测指标

每 tick 或每季节采样：

- 全图总水量。
- 海洋泄出水量。
- 总土壤湿度。
- 总养分。
- 草本数量和总生物量。
- 木本数量和总生物量。
- 动物数量。
- 动物出生数。
- 动物死亡数。
- 动物平均能量。
- 动物平均水分。
- 河流格子数。
- 湖泊格子数。
- 荒漠化格子数。

空间结构指标：

- 河流连通分量数。
- 最大水系长度。
- 湖泊连通域数量。
- 湖泊面积分布。
- 动物种群连通分量数。
- 局部灭绝区域数。
- 草本邻接聚集度。
- 木本邻接聚集度。
- 动物到最近水源距离分布。
- 冬季幸存动物到最近木本距离分布。
- 各平原口袋的动物数量曲线。

没有这些指标，难以判断系统是在真实涌现，还是参数偶然制造了表面现象。

## 19. 可视化与调试 UI

Eco-CA 的 UI 不只是展示层，也是调参和验证工具。第一版 UI 应优先服务于理解局部规则和发现失控。

### 19.1 地图图层

必须支持图层开关：

| 图层 | 表达 |
| --- | --- |
| `H` | 高度色带、等高线或阴影 |
| `B` | 基础地貌 |
| `S` | 河流、湖泊、冰、荒漠等地表覆盖 |
| `W` | 地表水热力图 |
| `M` | 土壤湿度热力图 |
| `N` | 养分热力图 |
| `F` | 水流方向和流量箭头 |
| `P` | 草本、木本、生物量 |
| `A` | 动物密度、个体位置 |

半三维表达至少应包含：

- 高度阴影。
- 等高线。
- 水流方向箭头。
- 坡度或通行成本提示。

### 19.2 单格检查器

点击任意格子时显示：

- 坐标。
- `H/B/S/W/M/N/F`。
- 植物类型、成熟度、生物量、压力。
- 动物列表。
- 最近一次状态变化原因。
- 当前 tick 的流入、流出、蒸发、渗漏。

### 19.3 时间控制

必须支持：

- 暂停。
- 单步 tick。
- 倍速播放。
- 回放最近 N 个 tick。
- 跳到季节边界。

### 19.4 调参面板

- `stable_default`
- `wet_world`

第一版可允许暂停时修改参数。若支持运行时热更新，应记录变更时间点，避免指标曲线无法解释。

### 19.5 调试叠加层

推荐支持：

## 20. 最小验证场景

每个阶段都应有固定 seed 的 deterministic scenario。场景用于判断规则是否可计算、稳定、可复现。

### 20.1 斜坡入海

地图：

- 左上高山。
- 右下海洋。
- 中间连续坡面。

验证：

- 水从高处流向海洋。
- 总水量满足质量平衡。
- 海洋泄出量可统计。
- 不生成不合理湖泊。

### 20.2 封闭盆地成湖

地图：

- 周围高地。
- 中央低洼平地。
- 无海洋出口。

验证：

- 中央形成湖泊。
- 湖泊面积在蒸发和泉眼输入之间达到稳定。
- 湖泊不会无限扩张到高地。

### 20.3 盆地缺口外溢

地图：

- 封闭盆地一侧有低缺口。
- 缺口外连接海洋方向坡面。

验证：

- 湖泊水位达到阈值后从缺口外溢。
- 外溢路径长期形成河流。
- 形成“河流-湖泊-出湖河”结构。

### 20.4 河谷草场

地图：

- 一条稳定河流穿过平原。
- 初始少量草本。

验证：

- 河岸 `M` 和 `N` 高于远离河流区域。
- 草本沿河谷形成高密度带。
- 非河岸区域仍可存在低密度草本，而非全图同质化。

测试状态：

- 已加入 `riverValleyGrassland` 的 M2.2 水文与草本验证用例。
- 用例验证地形/初始状态确定性、连通河流信号、河岸与远离河流区域的相对 `M`/`N`/草本生物量差异，并覆盖导出的河岸指标。
- 断言使用近河岸、中距离、远离河流区域的相对关系，不绑定固定常数，便于后续调参。
- 已加入 M3.1 木本验证用例，复用该场景检查木本状态确定性、生境排除和低山/山麓木本信号。

### 20.5 山脉隔离种群

地图：

- 中高山把平原切成两个口袋。
- 两边初始动物数量不同。

验证：

- 动物不能穿越 `H >= 3`。
- 两个口袋的种群曲线独立演化。
- 一侧可以灭绝而另一侧存活。

### 20.6 山麓庇护预备场景

地图：

- 平原草场连接低山林带，并保留稳定河流/河岸草带。
- 低山和山麓有稀疏木本初始斑块。
- 场景先服务 M3.2 植被稳定和动物前置指标验证，真正动物分布与死亡筛选留到 M4/M5。

验证：

- `foothillShelter` 场景已注册，并且同 seed 初始状态确定性一致。
- 720-1080 tick 长周期运行后，木本能跨越多个冬季继续存在。
- 低山木本覆盖不能扩张到全部可种植格，应保持为局部林带信号。
- 河流/河岸草本仍应存在，避免木本或季节循环使河岸草场消失。
- 动物前置/庇护指标非负，且木本庇护信号大于 0。
- 冬季木本格动物死亡率低于无木本格。
- 幸存动物到木本距离显著低于死亡动物。
- 春季幸存者能触发局部种群恢复。

测试状态：

- M3.2 已增加 `foothillShelter` 验证用例，覆盖场景注册和确定性、长周期植被稳定、河岸草本保留，以及 `woodyShelterCells` / `winterShelterCells` 等动物前置庇护指标。
- 当前用例仅验证动物规则上线前需要的植被和指标输入，不断言动物死亡率或繁殖恢复。

## 22. 设计结论

Eco-CA 的核心不是让个体变聪明，而是让环境场足够有结构。

动物只看 8 格并不削弱系统表现。只要湿度、植被、地势、林带和死亡筛选形成连续局部信号，远距离趋水、河谷聚集、冬季进林和区域隔离都可以从局部行为中涌现。

因此，本系统的优先设计方向是：

- 保持个体规则简单。
- 强化环境层之间的因果关系。
- 使用同步更新保证可复现。
- 通过指标曲线持续校准参数。
- 避免用宏观硬编码替代局部反馈。
